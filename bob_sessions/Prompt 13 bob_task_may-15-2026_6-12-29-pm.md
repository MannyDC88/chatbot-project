 5 | ## Project Overview

 8 | ## Critical Non-Obvious Patterns
 9 | 
10 | ### API Architecture
11 | - **Dual-purpose Flask server**: server.py serves both static files (index.html, etc.) AND the /chat API endpoint
12 | - **SSE streaming required**: /chat endpoint uses Server-Sent Events (text/event-stream), NOT standard JSON responses
13 | - **Response format**: Stream yields `data: {json}\n\n` lines, ends with `data: [DONE]\n\n`
14 | - **API key location**: Must be in .env file (ANTHROPIC_API_KEY), NOT in config.js (config.js is legacy/unused)
15 | 
16 | ### FAQ Knowledge Base Integration

19 | - **System prompt injection**: FAQ content is dynamically built into the system prompt using `build_faq_section()`
20 | - **Categories**: AC Repair, Maintenance, Pricing, Scheduling, General, Emergency Service

23 | 
24 | ### Frontend Streaming Implementation
